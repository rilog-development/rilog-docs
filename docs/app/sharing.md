# Sharing подій

Будь-яка подія в Rilog може бути поділена як **публічний snapshot** — посилання лише для читання, яке будь-хто може відкрити в браузері без акаунту Rilog.

## Створення посилання для sharing

1. Відкрийте з'єднання і знайдіть подію, якою хочете поділитися.
2. Натисніть кнопку **Share** (іконка посилання) в колонці Actions.
3. Rilog генерує публічний URL. Скопіюйте його до буфера обміну.

Сторінка sharing показує повні деталі події — заголовки, тіло, stack trace, мітку часу — у тому самому форматованому code view, що й у застосунку, але без елементів керування редагуванням чи навігацією.

## Варіанти використання

**Звіти про баги** — замість вставки стіни JSON у GitHub issue, поділіться посиланням Rilog. Ваш backend колега може побачити точний запит і відповідь без жодного налаштування.

**Code review** — поділіться реальним HTTP обміном як контекстом при перегляді зміни API. "Ось до" + "ось після" робить review конкретним.

**Командні debugging сесії** — вставте посилання sharing у Slack. Всі учасники дивляться на одні й ті самі дані.

**Тікети підтримки** — якщо у вас є внутрішні інструменти підтримки, зробіть пряме посилання на подію Rilog, щоб інженери підтримки не мали відтворювати проблему.

## Публічні vs приватні

За замовчуванням посилання sharing є **публічними** — доступними для будь-кого з URL, без автентифікації. URL є довгим випадковим токеном (не вгадується), тому практично є приватним з точки зору виявлення, але ставтеся до нього як до секретного посилання.

Посилання sharing показують:
- Тип події, мітку часу та статус
- URL запиту, метод, заголовки, тіло
- Статус відповіді, заголовки, тіло
- Stack trace (для error подій)
- Будь-які коментарі до події

Посилання sharing **не** показують:
- Інші події з того самого з'єднання
- Інформацію про акаунт
- Аналітичні дані

## Коментарі

Ви і ваша команда можете залишати коментарі до будь-якої події — включаючи shared події. Коментарі видимі для всіх, хто має посилання sharing, що робить їх легким способом анотувати те, що відбувалося ("ця помилка з'являється лише в Safari 17 на iOS") або відстежувати статус вирішення.

Щоб додати коментар:
1. Відкрийте подію (у застосунку або через посилання sharing).
2. Введіть текст у поле для коментарів на правій панелі.
3. Натисніть **Post**.
