# З'єднання

**З'єднання** (connection) створюється автоматично, коли екземпляр rilog-lib вперше надсилає подію з певним App Key. Ви не створюєте з'єднання вручну — вони з'являються щойно ваш застосунок запускається.

## Що таке з'єднання?

Думайте про з'єднання як про запущену сесію вашого застосунку. Кожного разу, коли ваш застосунок стартує і надсилає першу подію, Rilog фіксує з'єднання з метаданими про браузер, пристрій і середовище.

Один застосунок може мати багато з'єднань — наприклад, якщо 50 різних користувачів одночасно запускають ваш застосунок, сесія кожного користувача є окремим з'єднанням.

## Список з'єднань

На сторінці застосунку ви бачите список всіх з'єднань. Кожен запис показує:

- **Назва / ID з'єднання** — автоматично згенерований ідентифікатор (або мітка `environment`, яку ви налаштували)
- **Останнє оновлення** — коли була отримана остання подія
- **Успішні запити** — кількість HTTP подій з кодом 2xx
- **Невдалі запити** — кількість HTTP подій з кодом 4xx / 5xx
- **Браузер** — визначено з user agent

Натисніть будь-яке з'єднання, щоб відкрити його лог подій.

## Деталі з'єднання

Всередині з'єднання ви бачите:

- **Зведену статистику** вгорі (success rate, кількість помилок, час останньої активності)
- **Таблицю подій** з усіма подіями цього з'єднання, найновіші першими
- **Фільтри** для звуження за статусом, методом або типом події
- **Пагінацію** для з'єднань з великою кількістю подій

## Фільтрація подій

Використовуйте панель фільтрів над таблицею подій:

| Фільтр | Варіанти |
|---|---|
| **Статус** | Всі, 2xx Success, 4xx Client Error, 5xx Server Error |
| **HTTP Method** | Всі, GET, POST, PUT, DELETE, PATCH |
| **Тип події** | Всі, HTTP, Error, Page View, Custom |

Фільтри комбінуються (логіка AND). Наприклад: `Статус: 5xx` + `Метод: POST` показує лише невдалі POST запити.

## Очищення даних з'єднання

Ви можете очистити всі події з'єднання, натиснувши **Clear Data** у правому верхньому куті. Корисно для скидання тестового або staging з'єднання перед демо.

:::caution
Очищення даних є безповоротним. Воно видаляє всі події для цього з'єднання та не може бути скасоване.
:::
