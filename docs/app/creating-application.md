# Створення застосунку

**Application** у Rilog відповідає одному з ваших проєктів. Наприклад, якщо у вас є SaaS продукт, ви створюєте один Rilog застосунок для нього. Якщо є ще й маркетинговий сайт — це може бути другий застосунок.

## Кроки

1. **Увійдіть** на [rilog.online](https://www.rilog.online).
2. Натисніть **Create Application** (або кнопку **+**) в Application Center.
3. Введіть назву для вашого застосунку. Використовуйте щось описове — ви будете бачити цю назву в дашборді та списках подій.
4. Натисніть **Create**.

Вас буде перенаправлено на сторінку деталей застосунку, де ви зможете побачити свій **App Key**.

## App Key

App Key — це ідентифікатор, який ви передаєте до `Rilog.init()` у своєму frontend коді. Виглядає так:

```
rl_live_a7f3c8d2e1b4a9f0c3d7e2a1b5c8d9e0
```

:::tip Зберігайте в безпеці
Хоча App Key є клієнтськими обліковими даними (він з'являється в мережевих запитах браузера за визначенням), не записуйте його безпосередньо у вихідний код. Використовуйте змінні середовища — дивіться [Встановлення](/docs/rilog-lib/installation#environment-variables).
:::

## Кілька застосунків

Ви можете створити стільки застосунків, скільки потрібно. Поширені патерни:

- **Один застосунок на проєкт** — найчистіше розділення даних.
- **Один застосунок, кілька з'єднань** — корисно, якщо ви хочете бачити всі середовища (dev, staging, prod) в одному місці, але з різними мітками через параметр конфігурації `environment`.

## Видалення застосунку

Перейдіть до налаштувань застосунку та натисніть **Delete Application**. Це безповоротно видаляє всі пов'язані з'єднання та події. Скасувати не можна.

:::danger
Видалення застосунку негайно анулює його App Key. Будь-які запущені екземпляри rilog-lib, що використовують цей ключ, припинять надсилати події.
:::
