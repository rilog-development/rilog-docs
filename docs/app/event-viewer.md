# Перегляд подій

Event Viewer — це місце, де ви детально перевіряєте окремі події. Натисніть на іконку **розгортання** (або на сам рядок) у будь-якій події в таблиці з'єднання, щоб відкрити панель деталей.

## Колонки таблиці подій

| Колонка | Опис |
|---|---|
| **Статус** | Бейдж HTTP коду статусу (зелений для 2xx, червоний для 4xx/5xx) |
| **Метод** | HTTP метод (GET, POST тощо) |
| **Запит** | Шлях endpoint |
| **Час запиту** | Коли запит був зроблений |
| **Заголовки** | Попередній перегляд заголовків запиту |
| **Дані запиту** | Попередній перегляд тіла запиту |
| **Час відповіді** | Коли відповідь була отримана |
| **Дані відповіді** | Попередній перегляд тіла відповіді |
| **Дії** | Копіювати, Поділитися тощо |

## Розгорнуті деталі події

Натискання на рядок розгортає його на місці та показує повні деталі:

### Вкладка Headers (Заголовки)

Відображає всі заголовки запиту і відповіді у форматованому JSON. Ви можете скопіювати весь блок заголовків за допомогою іконки копіювання.

```json
{
  "Authorization": "Bearer eyJhbGc...",
  "Content-Type": "application/json",
  "X-Request-Id": "req_7f3a9b"
}
```

### Вкладка Request Data (Дані запиту)

Показує розібране тіло запиту. Для JSON payload — форматовано і з підсвіткою синтаксису. Для FormData поля показані як пари ключ-значення.

### Вкладка Response Data (Дані відповіді)

Показує розібране тіло відповіді від сервера. Помилки тут (наприклад, `{ "error": "Unauthorized" }`) — це саме те, що повернув ваш backend, що дає вам повну видимість.

### Помилкові події

Для JavaScript помилок розгорнутий вигляд показує:

- **Повідомлення помилки**
- **Повний stack trace** з посиланнями на файли
- **Файл / рядок / колонка** де виникла помилка

## Копіювання даних події

Використовуйте кнопку **Copy** поруч із будь-яким заголовком розділу, щоб скопіювати цей розділ як JSON до буфера обміну. Це спрощує вставку в GitHub issue або Slack повідомлення.

## Навігація між подіями

Використовуйте іконки стрілок **←** і **→** всередині розгорнутого вигляду (або клавіші стрілок клавіатури), щоб переходити до попередньої або наступної події без закриття панелі.
