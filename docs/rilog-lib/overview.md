# rilog-lib

`@rilog-development/rilog-lib` — це клієнтська JavaScript/TypeScript бібліотека, яка підключає ваш веб-застосунок до платформи Rilog.

## Що вона робить

Після ініціалізації з вашим app key бібліотека працює тихо у фоні та:

- **Перехоплює HTTP-трафік** — обгортає `fetch` і `XMLHttpRequest` глобально, фіксуючи кожен запит: URL, метод, заголовки запиту, тіло запиту, статус відповіді, заголовки відповіді та тіло відповіді.
- **Фіксує JavaScript помилки** — підключається до `window.onerror` і `window.onunhandledrejection`, щоб фіксувати stack traces і контекстні дані для кожної необробленої помилки.
- **Відстежує навігацію** — слухає History API і події `popstate`, щоб логувати зміни маршрутів у SPA.
- **Буферизує та надсилає асинхронно** — події пакуються і надсилаються до Rilog backend у фоні, без впливу на головний потік або досвід користувача.

## Що вона НЕ робить

- Не блокує запити або відповіді — лише спостерігає.
- Не надсилає дані синхронно — всі передачі асинхронні та не блокуючі.
- Не фіксує паролі або чутливі поля, якщо ви явно не налаштуєте це (і не варто).
- Не потребує жодних серверних змін у вашому застосунку.
