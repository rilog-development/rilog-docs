# Що таке Rilog?

**Rilog** — це легка платформа спостережуваності для веб-застосунків. Вона складається з двох частин, що працюють разом:

- **rilog-lib** — JavaScript/TypeScript бібліотека, яку ви встановлюєте у свій застосунок
- **Rilog App** — вебдашборд на [rilog.online](https://www.rilog.online), де ви переглядаєте всі зібрані дані

Після підключення Rilog автоматично фіксує HTTP-запити й відповіді, JavaScript помилки, перегляди сторінок та взаємодії користувачів. Ви бачите все в реальному часі в дашборді — без написання власної backend-інфраструктури.

## Чому Rilog?

Традиційні інструменти моніторингу або надто складні (потребують агентів, налаштування інфраструктури або DevOps-експертизи), або надто прості (фіксують лише необроблені виключення). Rilog знаходиться посередині: простий у налаштуванні, але достатньо потужний, щоб дати вам реальні відповіді, коли щось іде не так у production.

**Головна проблема, яку він вирішує:** ви деплоїте оновлення, користувач повідомляє про баг, і ви не знаєте, який API-запит провалився або що сервер насправді повернув. З Rilog ви відкриваєте дашборд, фільтруєте за часом або кодом статусу і бачите точний запит — заголовки, тіло, відповідь — за секунди.

## Основні поняття

### Connection (З'єднання)
**Connection** — це один запущений екземпляр вашого застосунку, підключеного до Rilog. Кожне з'єднання має унікальний ключ (ваш `appKey`). Ви можете мати кілька з'єднань на один проєкт — наприклад, одне для staging і одне для production.

### Event (Подія)
**Event** — це будь-яка зафіксована одиниця даних. Події мають тип (HTTP-запит, помилка, перегляд сторінки тощо), мітку часу та пов'язані дані (заголовки, тіло, stack trace тощо).

### Application (Застосунок)
**Application** у Rilog App — це логічне групування з'єднань. Ви створюєте застосунок у дашборді, отримуєте API ключ і використовуєте його в `rilog-lib`.

## Огляд архітектури

```
Ваш веб-застосунок
  └── rilog-lib (встановлено як npm пакет)
        ├── Перехоплює fetch / XMLHttpRequest
        ├── Фіксує JS помилки (window.onerror)
        ├── Відстежує переходи між сторінками (History API)
        └── Надсилає події → Rilog backend API

Rilog backend
  └── Зберігає події, будує аналітику

Rilog App (rilog.online)
  └── Відображає події, фільтри, аналітику, sharing
```
